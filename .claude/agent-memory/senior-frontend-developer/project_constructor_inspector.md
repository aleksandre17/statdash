The Constructor MVP core (C1 Inspector + C2 store unification) lives in `platform/apps/panel`.

**C1 — Inspector** (`platform/apps/panel/src/inspector/`): consumes the existing PropSchema seam (`nodeRegistry.getSchema(type)` from `@geostat/react/engine`) and renders a property panel generically — no per-type UI. Key seam = `FieldControlRegistry` (PropFieldType → React control, OCP). `LocaleField` authors a COMPLETE LocaleString record (one input per active locale, V13/V14 gold completeness). `EnumRefField` = data-driven options from the store. The engine also ships its own `PropSchemaForm` (`packages/react/src/components`) — app-agnostic, single-locale; the panel deliberately built a richer panel-local Inspector instead (multi-locale, grouped, enum-ref, store-bound) so engine/react stays agnostic (Law 3).

**C2 — unified store** (`types/constructor.ts`): killed the closed `CanvasNodeKind` enum. `CanvasNode = { id, type:string, variant?, props, childIds }` (was `{kind, config, children}`). `canvasPageAdapter.ts` has BOTH directions: `toNodePageConfig` (flat→tree, save/render) and `fromNodePageConfig` (tree→flat, load). API persistence (`lib/api.ts`) now stores a real NodePageConfig tree (was `{nodeIds,nodes}`).

**Why:** ADR (Constructor Phase 2) — the PropSchema inspector seam existed but had NO consumer in apps/panel; a closed node enum made newly-registered types unstorable (Law-1/OCP violation).
**How to apply:** New field type = register a control in `FieldControlRegistry`, Inspector unchanged. New node type = a new schema, panel unchanged. Round-trip invariant `fromNodePageConfig(toNodePageConfig(x)) ≡ x` is a fitness test in `canvasPageAdapter.test.ts` — keep it green.

**Phase C — chrome Inspector consumer + author→render e2e (landed):** The Inspector no longer hardcodes `nodeRegistry`; schema resolution is inverted behind a `SchemaSource` port (`inspector/schemaSource.ts`) — `Inspector` takes optional `schemaSource` (default `nodeSchemaSource`, preserving all node/panel behavior). `chromeSchemaSource` reads `chromeRegistry.getMeta(slot,key).schema`. ONE generic Inspector serves node/panel/chrome/control — a new slice kind = a new source, Inspector unchanged (OCP/DIP). Chrome is modeled as a CanvasNode (type=slot, variant=key, props=per-slot config); `ChromeInspectorPanel` adapts the selection + writes edits to `site.chrome[slot].config` via `updateChromeConfig`. `ChromePalette` lists registry chrome slots (capability discovery). Store: `site.chrome: Record<slot, ChromeSlotConfig>` (mirrors engine `SiteManifest.chrome`, serializes straight to config API), `chromeSelection` UI state (mutually exclusive with `selectedNodeId`), chrome actions extracted to pure reducers in `store/constructor.chrome.ts` (store hit the 400-line bloat ceiling — split by concern). The author→render MVP fitness is proven zero-code in `save/authorRender.e2e.test.tsx`: author via store+patchProp → saveGuard 4 checks → toNodePageConfig → render through real `NodePageRenderer` (from `@statdash/react/engine`, same path as CanvasView) → assert authored text renders; plus round-trip identity + block-on-incomplete-locale. **Gotchas:** (1) `inner-page` root requires `chromeConfig` on SiteProvider (min `{logoUrl:'',logoAlt:{ka,en}}`) or it node-errors. (2) The panel render path needs an `IntersectionObserver` shim (SectionNavContext scroll-spy) — added to `apps/panel/vitest.setup.ts` alongside the existing ResizeObserver shim. (3) `HeroDefaults.subtitle={ka:'',en:''}` is an empty-but-present LocaleString → saveGuard's "present but incomplete" check flags it; authoring must fill or clear it.

**In-flight engine dependency (verify before relying):** the prompt's ADR path `platform/.claude/agent-memory/architect/adr_constructor_phase2.md` does NOT exist on disk. The engine agent's slice-type additions are mid-migration: `packages/react/src/engine/types/index.ts` already re-exports `PropFieldSource` + `PropSchemaForm` already requires `'enum-ref'` in its renderer Record, but `slice-meta.ts` (the actual type source) has NOT added `enum-ref`/`coverage`/`PropFieldSource` yet — so `tsc -b` over packages/ has pre-existing errors there (not in panel src). The panel Inspector uses DEFENSIVE reads (`field as {coverage?}`, `field as {source?}`, `'enum-ref' as key`) so it compiles + works regardless. When the engine lands those, the defensive casts can be dropped; the `'enum-ref'` registry key already matches by runtime string.
