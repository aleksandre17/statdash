`docs/architecture/proposals/SPEC-deep-authorability-completion.md` (I am sole author, 2026-07-13). Owner named the felt gap: the object-model reform built the SUBSTRATE (Part grammar live) but the AUTHORING surface can't manage EVERY element's ENTIRE contract from the panel. Four felt gaps: chrome, full-functionality, data-pipe-without-metric, per-element style.

**The ONE reframing finding (verified in code on feat/ar49-m0-metric-first-authoring):** every element has TWO orthogonal declared-contract surfaces. (1) The **CONSTITUENT axis** — "what PARTS does it contain" — declared as `PartField`s, projected by `element.schema` band-item path. COMPLETE (ADR-041 Ph.1–6 landed; chrome/filters/kpi = 3 residence adapters of one port). (2) The **FACET axis** — "what universal CAPABILITIES (style/data/events/visibility) does it expose" — declared as TypeScript STRUCTURE on the config type (`NodeBase.view.styles: NodeStyles` node.ts:53, `NodeBase.data: DataSpec` :90, `NodeBase.on: NodeEventHandler[]` :127, transforms/vars/dataLinks, `ChromeSlotConfig.variant/region/order`) but NOT as a projectable AUTHORING contract. The generic dock (`dockSectionRegistry` + `FieldControlRegistry`) projects only `meta.schema` (per-type props); the universal facets fall out — hand-wired one-offs (`element.data`=metric-bind only, `element.visibility`) or absent (style/events/pipeline/chrome-variant). **All 4 gaps = ONE root cause: generic-projection ideology (ADR-038) applied to the part axis, never to the facet axis.**

**4 diagnoses (classified):** Gap1 chrome = [missing structural-facet contract] — `chromeParts` adapter (bandSource.ts:204) projects config-only; variant/region/order + site-frame's own composition unauthorable (site-frame `selectable:false`, Back deselects). Gap2 full-func = [missing contract] the umbrella. Gap3 data-no-metric = [buried] editors exist (`QuerySpecEditor`/`PipelineBuilder`/`CalcBuilder`) but Steward/Model-gated (FF-AUTHOR-NO-QUERY) + DataSpec not in FieldControlRegistry (→ SummaryCard glance). Gap4 style = [genuinely unbuilt] render ready (`resolveStyle`/`view.styles`) but no declared style contract (styleKeys private BEM), no StyleField, no element.style section.

**Canonical model:** `inspect(el) = projectParts (done) ⊕ projectFacets (this SPEC)` — 6 generic projections: content/params · STYLE · DATA(pipe/calc, metric-optional) · EVENTS · VISIBILITY · CHROME. Add a `FacetDescriptor { id, appliesWhen(meta), contract, readPath }` registry; derive dock sections from it (reuse `dockSectionRegistry`, OCP). appliesWhen reads a cap/declared-field, never a type. Thin-base honored: facet contract declared ONCE at platform, element opts in by cap/field (no base bloat). Style/events need declared contract ADDED (parts manifest + token catalog; trigger vocab over existing NodeAction union); data needs wire+ungate (editors exist); chrome needs structural-facet + site-frame inspector.

**Ideology verdict:** generic-projection HOLDS + canonical — applied to only 1 of 2 axes. Best form = name Parts ⊥ Facets, both generic+declared; elevate ROOT-4 (Facet) from render-side opt-in to projectable authoring contract. Caveat (refinement not rejection): rich facets (DataSpec/NodeStyles) get rich registered controls via FieldControlRegistry Strategy seam — genericity is in DISPATCH, not auto-gen. Benchmark: Webflow/Framer/Builder/Figma all do "select any element → fixed set of facet tabs (style/settings/data/interactions) opted-in by declaration" — our facet registry is the canonical form + declared-data moat.

**Recommended FIRST slice:** per-element **StyleField** (Gap4) — genuinely unbuilt, most visible (every element), no role/policy entanglement, design converged ([[project-framework-grade-style-system]] S0/S1: new PropFieldType 'style', activate `enum-ref source:'tokens'` vs TOKENS_CATALOG, whole-element view.styles MVP no parts). Doubles as the facet-section seam Data/Events reuse. Then S2 data(lens per D-DA1) · S3 events · S4 chrome · S5 completion gate `FF-EVERY-DECLARED-FACET-PROJECTED`.

**One-way flags (none hard):** D-DA1 data honesty boundary — keep FF-AUTHOR-NO-QUERY as a LENS not wall (author does transform/derive over governed source; raw-source define stays Steward) [rec lens]; D-ST1 token constraint = warn; D-CH1 site-frame inspector = yes; D-EV1 triggers from caps first (YAGNI).
