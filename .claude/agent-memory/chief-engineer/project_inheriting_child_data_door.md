**Composition fact (gdp page, config-proven via `/api/config/pages/<id>`):** ALL data is owned by SECTIONS (each an inline `query` DataSpec). Each section's child is a `wrap` whose chart+table VIEWS are DATA-LESS and render from the section's shared data. On gdp: 8 sections own data, 16 render nodes (8 chart + 8 table) inherit. Same pattern on regional/accounts.

**S2 — pipeline node → workbench EMPTY (deterministic).** The canvas frames the inner chart as a selectable node (synth id like `production-0-0`). Its inspector DATA facet (`DataFacetField.tsx`) reads `value = props.data = undefined` because the chart is data-less → shows `summary-unbound` + a door (`open-data-workbench`) that opens an unbound `freshPipelineSpec()`. The data-OWNING section exposes NO door (only `onboard-data-cta`). So the DATA-facet projection sits on the WRONG element in the containment hierarchy (Law 10 / ADR-041) — offered on the data-less child, absent on the data owner.

**S3 — visual VANISHES on exit (deterministic, in-session-only).** `DataFacetField.openWorkbench` (:108-126) runs `const seed = adoptOnOpen(spec); if (seed) onChange(seed)` ON OPEN. For a data-less child `spec===undefined` → `adoptOnOpen` (`dataFacetModel.ts:35-37`) returns `freshPipelineSpec()` → `onChange` WRITES empty `props.data` that SHADOWS the inherited section data → chart renders "მონაცემები არ არის / empty.desc" (leaked i18n key). A look-only gesture fabricates a no-data state (violates Law 11). Reload restores it (write hits in-memory page store; E0 `2fed78e7` deleted the auto-PUT so nothing persists).
