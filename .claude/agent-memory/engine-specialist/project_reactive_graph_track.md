**THE V2 CRUX — TWO GENUINE DIVERGENCES (block V3 render-safety; the deliverable, not papered over):** value-parity caught two "dep a static walk cannot see" classes extractDeps' ref-totality structurally misses:
- **A — val-based specs under-fire on ambient dims.** `timeseries`/`growth`/`row-list`/`ratio-list` read `storeVal`→`_val`→`_matchedValues(code, ctx.dims)` (store-impl.ts) = the WHOLE ambient coordinate, so their value depends on EVERY dim (geo etc), but `extractDeps` records only TIME_DIM for them → a non-time dim change serves a STALE memo (under-fire).
- **B — a `$d`/`$cl` row-label join is locale-dependent but unmarked.** The join tags a LocaleString label resolved per-locale at the boundary; `deps.locale` is set only for localized DISPLAY FIELDS, not row-label joins → a locale toggle changes rows with no edge (under-fire).
FIX DIRECTION (V3 precondition): widen the extractDeps dependency MODEL — val-specs must depend on the ambient coordinate (all active dims, or an explicit read-coordinate model), and a display-join must set `deps.locale`. Until then V3 is NOT render-safe.
