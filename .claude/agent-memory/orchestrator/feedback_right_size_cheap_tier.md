A real logistician sends the cheap tier (junior-executor/Haiku, Sonnet middles) into the SAFE, low-stakes zones to save cost — and NEVER into work that isn't theirs (judgment/architecture). Defaulting expensive Opus seniors onto everything (even mechanical work) is a logistics failure — cost burned with no quality gain.

**Safe zones for the cheap tier:** scripts, boilerplate, bulk pattern-replication across N files, templated test/fixture updates, doc/file relocations, mechanical renames/sweeps — anything with a clear spec and no real judgment.
**Never the cheap tier:** object-model/architecture decisions, new patterns, semantic/UX judgment, anything where a wrong call costs.

**The key money-saving pattern:** **a senior establishes the pattern ONCE (judgment) → a junior replicates it across N files (mechanical).** E.g. the MUI→Radix migration: senior builds the owned `Select` + the token-binding pattern; junior sweeps the remaining N MUI-swap files against that pattern. The mechanical layout sweep (Box/Typography/Stack → token primitives) is pure junior work.

**Why:** owner (2026-07-15): "you have Sonnet and Haiku in your team; a real logistician sends beginners to the simple zones to save cost, but never where it's not their job. Use them on scripts, some tests — safe zones. You're the leader, just don't misroute." The lead had been routing expensive seniors onto everything.

**How to apply:** FORECAST each task's decision-density before routing; safe/mechanical → cheap tier, judgment → senior/Opus. For bulk migrations, split: senior sets the pattern, junior replicates.
