The owner demands the maximum. The standing rule (stated 2026-06-27):
1. If a concept strengthens the platform's capabilities → adopt it **fully**, never partially.
2. **Proactively hunt** for powerful concepts we have NOT yet adopted (mine reference platforms) — do not say "no" to strong ones.
3. What we adopt, we **complete**: used on **every layer**, with **nothing left unused**.

**Why:** this is Law 4 ("the full benefit of standards, not partial") taken to its limit, plus Law 8 (platform thinking). A "cathedral without a congregation" (a built, fitness-locked capability with zero production consumers — e.g. the semantic-layer MetricDef registry with 0 registered metrics, the `scope.metric` perspective key authorable-but-not-wired) is a Law-4 VIOLATION, not an acceptable YAGNI deferral. Do NOT frame partial adoption as "conscious deferral / fine" — the owner explicitly rejects that.

**How to apply:**
- When the board/an agent flags "mechanism shipped, adoption pending" → that is REMAINING WORK at full priority, not a deferral. Plan the full every-layer wiring (contracts→core→charts→react→plugins→api→db→panel→provisioning) + a real consumer + a fitness function that PROVES it is used.
- The "complete = proven-used" discipline: a capability is done only when a fitness asserts a real runtime consumer exists ("no registered/authorable capability without a runtime consumer"). See [[green-gate-panel-typecheck]] sibling discipline.
- **The one critical boundary (still principled, serves the doctrine):** "adopt fully" ≠ "open every speculative door." An empty fitness-locked door is itself "unused" — so adopt a concept WITH its real consumer, never as speculative machinery. And keep separating genuinely-strengthening concepts from incumbent-specific cruft (principled refusal still applies to weak concepts; see [[guardian-of-canon]]).
- Removing a dead half-thing (e.g. the `custom` DataSpec discriminant with no resolver, a redundant 2nd extension path) IS a valid form of "nothing unused" — completion can mean delete, not only wire.
