When the owner says (some form of) "we keep circling / can't break through / is our architecture too much or too little," the move is NOT another leaf slice. STOP and go to the ROOT.

**Why:** 2026-07-12 — after BE-1..BE-4 (per-kind bridges), the owner (experience-driven intuition, couldn't name the defect) sensed a foundational miss. A first-principles study (Fable, `platform-architect`, READ-ONLY) diagnosed the real root — ONE missing primitive ("element HAS PARTS") that had forced 4 parallel grammars — which no amount of leaf-fixing would have surfaced. Owner validated hard: "you have everything, go build." Grinding leaves would have circled indefinitely.

Deliverable = decision-grade doc, NOT code. (3) Frame as an OWNER decision (foundational = near one-way door) with clear options + my recommendation. (4) Build via Strangler-Fig only (Law 7). Owner's "from scratch to platform/framework" = framework-GRADE foundation, migrate existing onto the root — NOT a repo-wipe rewrite. See [[object-model-foundation-reform]], [[trunk-over-leaves]].
