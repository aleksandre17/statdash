**Rule:** the lead MANAGES the agent workforce — he does not fling briefs and hope. Every launch is grounded in VERIFIED state, tightly scoped, cost-ceilinged, with a fast exit on a wrong premise; lead-owned instruments stay the lead's. Time and tokens have real cost; wasting them is a leadership failure, not the agent's.

**Why (2026-07-11):** the owner, frustrated: *"why did the agent change the registration .md file? It even REFUSED to do something — and how many tokens were spent. If it couldn't, it should have exited in time. Take the reins — you're the LEADER, not a pushover. Time and cost matter to you. Leaders behave like this. You're not managing the situation."* Two concrete failures, BOTH mine: (1) I launched an Opus audit agent to BUILD a backbone that already existed — it correctly REFUSED, but burned **176k tokens** discovering my brief was on a STALE, unverified premise. (2) I let a build agent edit `ARCHITECTURE-REGISTRY.md` — a LEAD-maintained instrument. The waste + the boundary-slip were my management, not the agents'.

**How to apply:**
1. **Verify-before-brief** — establish the real current state (read/grep/probe) BEFORE writing a brief. Never brief on a remembered/assumed premise. A brief built on sand makes the agent burn tokens discovering my error. [[model-launch-ledger]]
2. **Exit-fast + ceiling in EVERY brief** — explicit: "if the premise is wrong, or you're blocked, or the task is already done — STOP IMMEDIATELY and report the blocker in <N tool-uses. Do NOT explore/burn tokens." Give a scope/token ceiling. A right refusal is good; a 176k-token refusal is a briefing failure.
3. **Right-size model + scope** — match the tier to decision-density ([[worktrees-only-when-truly-parallel]]); do NOT launch a 200k-token Opus agent for what I can do myself or a Sonnet/Haiku can. Often the cheapest correct executor is ME, directly.
4. **Lead-owned instruments stay mine** — `ARCHITECTURE-REGISTRY.md` ("the lead MAINTAINS this") + the `work/` board are MY instruments. Agents do code/tests/design and REPORT; **I** record the registry/board rows. Do not delegate registry/board maintenance.
5. **Control + verify + own** — I manage the situation, independently verify every result ([[verification-fit-per-situation]]), and own the cost. Delegate as much as I judge right — but as a leader deciding, never as a pushover flinging work. Related: [[lead-decides-never-asks-tactics]].
