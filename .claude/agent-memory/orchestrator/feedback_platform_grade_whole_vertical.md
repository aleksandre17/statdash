The owner's standing quality bar (reaffirmed repeatedly 2026-06-28): **we CREATE → IMPROVE → USE (to the max) → every tier, with innovation and the highest concepts/standards/architectures.** Two binding parts:

**1. Whole-vertical, no tier second-class.** A capability is NOT done when it works only in the renderer. The SAME maximal bar applies to ALL three tiers:
- **renderer** (packages/core, react, plugins) — how it renders;
- **Constructor / panel** — how a non-engineer AUTHORS it with NO code (inspector controls, registries, validation visible in the authoring UI);
- **API / provisioning** — how it is persisted, served via the manifest, and validated (refs-exist, illegal states unrepresentable, fail-fast).
The bar itself: **truly platform-level, competitive with the strongest reference platforms (Tableau / PowerBI / Looker / Cube / Malloy / Vega-Lite / SDMX), forward-leaning, future-proof, flawless.** When dispatching any capability, require the whole vertical in the brief — never renderer-only.

**2. Foundational / IRREVERSIBLE decisions get special treatment.** Owner: "we have no way back, so everything must be ideal end-to-end; don't rush; be guided by proven systems/architectures/concepts and by HOW the teams who shipped them solved their problems." Process that worked this session (time-mode decoupling):
- do NOT rush, do NOT build first — a deliberate, read-only architect exploration + a critical red-team/synthesis pass (different lens) first;
- ground in real reference-platform concepts AND their problem-solving experience; survey, synthesize, reason across multiple logics (future-lens + problem-lens);
- the team DECIDES with senior conviction and brings ONE decided architecture (not a menu); the owner RATIFIES the single irreversible call (owner owns the no-way-back point; the team owns the engineering judgment);
- decide C-vs-B (adopt-now vs defer-behind-a-named-door) on the REAL DATA + the [[maximal-adoption-doctrine]] (no empty cathedral) — "maximal" = orthogonality / adopt-with-a-real-consumer, NOT speculative axis-count.

**How to apply:** hold every tier to this bar; Law 7 is absolute (existing code adapts to the plan, not vice-versa; many changes are acceptable; no degradation, no less agnosticism). See [[guardian-of-canon]] (decide with conviction, refuse degradation) and [[maximal-adoption-doctrine]] (use everything we create).
