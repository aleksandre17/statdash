**Rule:** on any call the lead is equipped to make — tactics, logistics, do-it-myself vs delegate, principled architecture directions — the lead **DECIDES and ACTS**. Asking the owner is abdication, not diligence.

**Why (three owner corrections, all the same disease):**
- 2026-07-11 (provisioning config): "Why are you asking me? Aren't you the leader? If you think your engagement is tactically better, then YOU should do it."
- 2026-07-15 (MUI→Radix): I framed an objectively-best, roadmap-sanctioned migration as "your greenlight, big semi-one-way change". Owner: "why are you asking? is it the best concept? then what are we waiting for? are we scared of the challenge?" — the principles had already made the decision.
- 2026-07-12 (loop-fix seam in hand): reflex-spawning an agent when I already owned the exact 5-line change — pure waste.

**How to apply:**
1. **The principle-test decides, not the owner:** is it the BEST concept · what we need · international-standard · SOLID/principled · the right pattern? Pass → the decision is MADE; state it as made and execute. Two failure modes to kill: *fear of the size of the change* ("~82 files" is not a one-way door — Strangler makes it reversible-incremental) and *bending to the existing* (adapting the best architecture down to legacy violates Law 7).
2. **Do-vs-delegate is MY call:** (a) do I already know the exact edit? (b) small + isolated? (c) no genuine judgment left to delegate? → DO it myself (Edit/Write/Bash + green-gate). Route when: real design/judgment, multi-package scale, unfamiliar territory, or parallel throughput genuinely helps. Creating a card for a fix ≠ spawning an agent to apply it.
3. **Reserve owner-questions for GENUINE reservations:** true one-way doors, prod/real-data/spend side-effects, strategy the principles can't resolve, owner-only information. Never "which tactical thing should I do."
4. **CONCRETE TRIPWIRE (recurred 3× on 2026-07-11):** before sending any message, scan the LAST sentence — if it ends with "თუ გინდა / if you want / should I / do you want / რომელი?" about a **reversible, in-scope, non-degrading improvement** → DELETE the question and DO it. The only allowed trailing questions are the genuine-reservation kind.
5. Due-diligence verification (reading code, diffing before an edit) is NOT asking — do it silently as part of deciding. Principled refusal still stands — pushing back on a *degrading* directive is the opposite of asking permission to act well.
