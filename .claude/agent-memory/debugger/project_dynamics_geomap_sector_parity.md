The LAST three OLD→NEW (API-arch) render-parity gaps on geostat, all LIVE-ONLY (green tests). Root causes + fixes (file:line in platform/), verified by deploy→screenshot vs OLD (:5171, `national-accounts-app`; NEW geostat :3002, api :3001 internal). The OLD app's bundled config/logic was the ground-truth oracle — its minified defaults (`is=String(ns[ns.length-1])`=max year) revealed intent the NEW provisioning had lost.

**GAP A — dynamics timeseries collapses to ONE bar (gdp/accounts/regional).** TWO compounding causes, both surfaced only on the async ApiStore (the OLD in-memory store masked them):
1. `toYear` default in `apps/api/provisioning/geostat.provisioning.json` was `{from:'options',pick:'last'}` but the toYear options pipe sorts `dir:'desc'` → `pick:'last'` = `rows[length-1]` = the MINIMUM year (2010), not max. OLD used a literal computed `String(ns[ns.length-1])`=max. Fix: `toYear` default `pick:'last'`→`'first'` (desc-sorted → first = max). fromYear (asc+first=min) was already correct.
2. The bigger one: in range/dynamics mode the `year` param's default RE-RESOLVES after the mode effect clears it (`useFilterState` ran `resolveDefaults` over ALL bars' params regardless of bar visibility) → `ctx.dims['time']` re-pinned to a single year → the QueryResolver's `rangeMode = isUnsetTime(time)` is FALSE → `extractRequirements` warms only `[time]`, the unbounded read never fires → on the async store the chart sees ONE warmed year. Fix: `packages/react/src/filters/useFilterState.ts` — gate default resolution by BAR VISIBILITY (`evalWhen(bar.showWhen, state)`); a param in a bar hidden by the current mode gets NO default. Range mode → year-bar hidden → `time` stays unset → unbounded read → full span. (Tell: live wire showed per-year `from=2010&to=2010` reads + NO unbounded `?dataset=...&filter=...` request.)

**GAP B — regional choropleth map renders BLANK.** TWO causes:
1. `/data/georgia-regions.geojson` returned the SPA index.html (2240b nginx fallback) — the asset was NEVER in the repo. OLD shipped it (580837b, 13 features, `shapeISO` GE-AJ…GE-TB matching the geoCodeMap). Fix: copied it to `apps/geostat/public/data/georgia-regions.geojson` (Vite serves public/ → root). (The OLD GeoMap is byte-identical flat-fill + tooltips, NO value choropleth — "colored by GVA" was an imprecise report; adding the asset reproduces OLD exactly.)
2. Even with the asset, the map box was height 0 (13 paths rendered into a 0px Leaflet container → blank + `Invalid LatLng (NaN,NaN)` from a degenerate FitBounds). Cause: geograph renders as a `.panel` (not a `.section`), and the node's `view.styles.height:'16:9'` token lands on the OUTER `.panel-col` — a flex column whose computed `aspect-ratio` is `auto` (flex columns ignore it). `packages/react/src/styles/panel-layout.css` had a height flex-chain ONLY for `.panel-col > .section …`, none for `.panel`. Fix (CSS-only): added `.panel-col > .panel`/`.panel__body` flex chain + re-homed the aspect-ratio from `.panel-col[data-height]` onto `.panel-col[data-height] > .panel > .panel__body` (the real content box). leaflet-container → 295px. (Reverted an earlier attempt that passed `vs.body` data-height onto the chart-wrap — it conflicted with `[data-height] .chart-wrap{height:100%}`.)

**GAP C — regional sector donut "No data".** The donut query pins `sector:{$ne:'_T'}` (break down by sector) but ctx.dims['sector']='_T' (the "all sectors" select default). `ApiStore.toObsParams` pre-seeds `filterRecord[dim]=ctx.dims[dim]` for every nonTimeDim FIRST, then a pure `{$ne}` (no `$ctx`) did NOTHING (intending "send no pin") — leaving the stale `sector:'_T'` on the wire. So the fetch was scoped TO `_T`, and the client-side `matchesFilter` `$ne:'_T'` then dropped EVERY row. Fix: in the pure-`$ne` branch, `delete filterRecord[dim]` so q.filter intent wins over the ctx baseline (fetch broad, exclude `_T` client-side). Extracted the whole filterRecord assembly to `buildObsFilterParam` in `data/store-filter.ts` (store-api.ts hit the 400-line ceiling). Regression: `apiStore.async.test.ts` "pure $ne DROPS a stale ctx-baseline pin".

**Class / how to apply:** mirrors [[apistore-val-measure-drop]] (a per-spec pin honored by the sync store but dropped/defeated on the async store) and [[transform-registry-treeshake-and-etag-slice]] (live-only, green-test). When debugging async-store render gaps: (a) dump the OBS WIRE requests (`?dataset=…&from=…&to=…&filter=…`) — a missing unbounded read, a wrong from/to, or a stale ctx pin defeating a `$ne`/override is the tell; (b) the OLD bundle's (even minified) config defaults are the parity oracle — resolve its vars; (c) filter DEFAULTS must respect bar visibility (don't cross-pin a hidden mode's param); (d) `.panel` nodes need their own height flex-chain — aspect-ratio on a flex `.panel-col` is `auto` (no-op). All 1066 core+plugins+react tests green after each change.
