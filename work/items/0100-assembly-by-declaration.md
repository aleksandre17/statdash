**The root (two-lens study, synthesized).** The panel still assembles an object by **dispatch** (a `switch(type)` picks the editor) and **by hand** (a dropped tile is a blank shell the author must hunt to fill). Both are the same defect at two levels — assembly is *not declaration-driven* — and both map 1:1 to the owner's four complaints: (#1) not loosely-coupled, (#2) concept↔logic entangled, (#3) assembly hard for a non-expert, (#4) built capabilities not leveraged.

**Primitive 1 — DataSpec authoring-contract registry (the *binding* axis gets its port).** Object assembly has three axes: identity/props, containment (ADR-041 Part port), and **binding** (which data it resolves). Axes 1–2 are declaration-driven; axis 3 never got its port. `DataSpecEditor` hand-wires three parallel `switch(type)` (SpecBody editor import · defaultSpec shape · SPEC_CATALOG stub) — the exact "external if type==X" the Bounded-Element law refuses. Fix: each bind-kind DECLARES its `make()` default + its authoring surface (a `PropSchema` **or** a registered rich editor — the rich modalities survive, resolved not switched). The two switches collapse into one generic renderer. Strangler anchor: transform-steps & filter-params already carry a `PropSchema` via `SchemaSource` — extend that proven pattern *up* to the DataSpec kind. Closes #1, #2. **Guard: `FF-NO-DATASPEC-SWITCH` + `FF-DATASPEC-AUTHORING-COMPLETE`, landed FIRST as a ratchet.**

**Primitive 2 — Composed-Preset projection (the object lands as a composed whole).** A preset = a partial element declaration (type + sensible props + bound DataSpec + pre-wired trend/threshold/visibility/style), registered on the SAME open registry, projected into the palette as an insertable whole — the "composed starting point" the whole reference class ships (Builder Blocks · Puck defaultProps · Form.io templates · Grafana viz-suggestions) and we lack. Rides on a substrate un-burying pass: drop the `DataFacetField:124` gate (all bind-kinds reach the DataWorkbench), wire `VisibilityBuilder` into the ~42 places it's absent, build the missing `TrendField` projection (raw JSON in ~33 places). Closes #3, #4 — the buried capabilities ship pre-wired *inside* each preset, not behind an expert escalation. **Config-as-SSOT clean (Law 2), one additive registry field, zero per-type special-case.**

**Why this order.** Primitive 2 STANDS ON primitive 1: a preset pre-fills a *bound DataSpec*, which the registry's `make()` makes declarable. So: registry FIRST (foundation + ratchet), composed-preset SECOND. Both additive, both Strangler, neither touches the engine object-model; ADR-049 EXTENDS ADR-038/041 — no fifth grammar.

- **P1 — DataSpec authoring-contract registry** — promote `spec-catalog` to a real contract (`make()` + authoring surface per kind); collapse the three `DataSpecEditor` switches into one generic renderer; rich editors register, not switch. Gate `FF-NO-DATASPEC-SWITCH` lands first as the ratchet. → build → gate → deploy :3013 → shown.
