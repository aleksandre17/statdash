**The international canon:** Grafana's Query Inspector (THE reference: every panel exposes the exact request JSON it fired) · Superset «View query» · Power BI Performance Analyzer. The transparency layer is a first-class debugging instrument for the data steward — it must ALWAYS answer "what exactly is being asked of the store?"

**Root (diagnosed):** `describeStewardDetail` was written for the query-shaped workbench (W-P2); W-P5b converted the workbench to pipeline-shaped and the steward derivation was never taught the new head forms → fail-soft EMPTY → an empty pre-block renders with no height → "открываешь и ничего" (the owner's exact catch). Double breach: stale derivation + an UNDECLARED empty (Law 11 in our own tool).

**The build:**
1. **One SSOT derivation:** the lowered wire read comes from the engine's `sourceHeadObs(head)` (core export, W-P5c — the exact derivation the read/warm use), NEVER a pane-local re-implementation: governed head → the resolved obs params (codes via resolveMeasureRef, dims, time — after metric-natural neutralization where applicable); steward head → its ObsQuery verbatim; inline head → NO wire read, declared.
2. **Every state declared:** inline → «wire-მოთხოვნა არ არის (inline rows)»; unbound → «მიაბი მეტრიკა — მოთხოვნა აქ გამოჩნდება»; a real read → pretty-printed, dark-mode-safe (the `action.hover`+`text.primary` pair, 8d86baf class). NEVER a zero-height void: the block carries a min-height + its empty text.
3. **Both blocks live:** raw DataSpec JSON (the emitted pipeline spec) + the lowered ObsQuery; per-step governed-noun consumption stays (E4 — this pane is also the EXPLAIN seam).
4. A11y: labeled regions, accordion keyboard, contrast per Law 9.

**Hard boundaries.** Steward-lens only (plane law) · no second lowering path (sourceHeadObs is the only source of wire truth) · apps-only unless `sourceHeadObs` proves insufficient for a head form — then EXIT-FAST and report the core gap, do not re-derive in the panel.
