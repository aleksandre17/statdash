**Intent.** The author must build a query as a VISIBLE, STEPPED PIPELINE ‚Äî seeing the raw data flow through every step and the resulting declarative query alongside ‚Äî while raw data itself gets ONE canonical home, separated from the semantic model and specs forever. Full power, simple perception, whole standards.

**Reference anchors (Law 4 ‚Äî adopt whole):** Power Query's step-pipeline with a live grid per step ¬∑ Grafana's builder‚Üîcode duality (the generated query always visible) ¬∑ Vega-Lite / Tidy-Data transform grammar (declarative verbs: filter/aggregate/derive/pivot) ¬∑ SDMX (our ObsQuery stays the wire truth) ¬∑ the W2 governed-noun spine (metrics/dims as vocabulary, never raw codes in the author plane).

**Design deliverables (this card's DoD):** a decided SPEC (not a menu): the pipeline grammar (declarative steps over the ONE evaluator/lowering path) ¬∑ the data-home IA (raw ‚Üí governed model ‚Üí specs ‚Üí elements as the visible floor plan; DQ expectations declared at the raw floor ‚Äî backlog rec #1 folds here) ¬∑ the authoring surface concept (step rail + live grid + generated-query pane; where it lives in the studio IA) ¬∑ Strangler route from today's tag-based editors ¬∑ wave decomposition sized for WIP=1 ¬∑ what is refused and why.
