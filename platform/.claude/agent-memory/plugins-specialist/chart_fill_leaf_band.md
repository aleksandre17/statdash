The "vertical bar frozen at ~176px in a stretched equal-height card" defect (owner headline, fixed in commit 84999e2). Durable because the DOM chain is non-obvious — the band lands on the LEAF, not a container.

**How the band reaches `.chart-wrap`:** a `section`'s child is a `wrap` node that declares `styles.aspectRatio` (e.g. geostat gdp expenditure `{default:"16 / 9", sm:"4 / 3"}`). `WrapShell` is a **transparent node — renders NO DOM of its own** (`packages/plugins/nodes/layout/wrap/default/WrapShell.tsx`), it only provides `WrapStyleContext`. `defineShell` floors that wrap style UNDER each descendant shell's own `view.styles` (child wins). So the aspectRatio lands on the CHART node's merged styles → `applyNodeStyles` emits the `data-aspect=""` presence flag (resolvers/node.ts only emits it when `aspectRatio` is set) → `Chart.tsx` spreads `bodyAttrs` onto `.chart-wrap` → the band is on `.chart-wrap[data-aspect]`, a DIRECT `>` child of `.section__view[data-view="visible"]` (SectionShell maps each child into a `.section__view` with `resolveViewState`).

**Why it froze:** node-styles.css base rule `[data-aspect] { height: var(--size-panel-height) }` pins the leaf at a FIXED band. The panel-layout.css **container** growable-band rules (`.panel-col > .section > .section__body[data-aspect] { flex:1 1 var(--size-panel-height); height:auto }`) only fire when the band is on `.section__body` — but here the body is bandless (`.section__body:not([data-height]) { flex:1 }`). So when an equal-height sibling (the taller paired donut) stretches the card, the frozen leaf can't grow → white gap + ApexCharts `height:'100%'` measures the frozen box.

**The fix lives in TWO places (both required):**
1. `packages/plugins/panels/chart/default/utils/apex/base.ts` — `BASE.chart.parentHeightOffset:0` (contribution/cartesian vbars already emit `chart.height:'100%'`; the offset:0 keeps the fill flush).
2. `packages/styles/src/css/node-styles.css` — a LEAF growable-band block mirroring the container model: `[data-view="visible"]:has(> .chart-wrap[data-aspect])` → fill-flex column; `[data-view="visible"] > .chart-wrap[data-aspect]` (+ ratio `data-height` aliases 16:9/4:3/1:1/21:9) → `flex:1 1 var(--size-panel-height); height:auto` (defeats the frozen base height). Plus `chart.css` `.chart-wrap` = flex column, `.chart-wrap__render` = `flex:1` → unbroken chain to the Apex mount div.

**How to apply:** any "chart doesn't fill / frozen plot / white gap in a card" report — first check whether the band is on the LEAF (`.chart-wrap[data-aspect]`, via a `wrap`'s aspectRatio) or on the CONTAINER (`.section__body[data-aspect]`). Leaf case → node-styles leaf block; container case → panel-layout.css. Horizontal bars are EXEMPT (they size by `categoricalChartHeight` → px per row on the ApexRenderer `height` prop, which overrides `options.chart.height`); donut/treemap roots are `height:100%`. **Guards:** `panels/chart/default/utils/apex/chart-fill.test.ts` (Apex config: `height:'100%'` + `parentHeightOffset:0` for vbar/contribution, ≥1024 responsive bps never numeric, categoricalChartHeight vbar='100%'/hbar=px) + `packages/styles/src/panel-sizing.fitness.test.ts` (CSS: the leaf growable-band rule text). See also [[apex-responsive-yaxis-formatter-drop]] and [[responsive-fix-wave]] (audit shots are main-rendered — but THIS defect is real on-branch too, confirmed by server-verify).
