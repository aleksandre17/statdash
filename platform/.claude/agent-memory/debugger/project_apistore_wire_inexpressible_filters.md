Two live-only (caps.sync=false) ApiStore defects, same root class as [[apistore-val-measure-drop]] — a per-spec scope honored by one code path but not its sibling.

**Bug 1 — KPI warm reintroduced a wildcard-deleted dim (range/dynamics kpi-strip cold-throw).** `useKpiRows` (`packages/react/src/engine/useKpiRows.ts`) built the warm ctx as `{ ...sectionCtx.dims, ...r.dims }`. `extractKpiRequirements` builds `r.dims = atTime(t, withFilter(ctx, filter)).dims`, which already DELETES a dim a KPI filter wildcards (e.g. a national-total KPI on a region-pinned page sets `geo:''` → withFilter drops geo). The spread REINTRODUCED `geo` from sectionCtx.dims → warm key had geo=R2, the read (storeVal → querySync, ctx from withFilter has NO geo) had none → cache miss → `ApiStore.querySync called cold` → NodeErrorBoundary "Failed to load component". Fix: warm under `r.dims` VERBATIM (`dims: r.dims`) — r.dims IS the authoritative resolved read ctx. Regression: useKpiRows.async.test.tsx "warms under the EXACT read dims when a KPI filter wildcards a page-pinned dim".

**Bug 2 — $ne (exclusion) String()-mangled onto the wire (empty regional map).** The regional `geograph` map filters `geo:{$ne:'_T'}` (every region except the national total). `ApiStore.toObsParams` only handled `$ctx` objects; a `{$ne}` object fell to the `else` → `String(fv)` = `"[object Object]"`, an unmatchable wire filter → 0 rows. The observations route (`apps/api/src/routes/stats/dim-filter.ts` + `filterSchema`) expresses ONLY `@>` containment + `= ANY` (array) — NO `<>`, and its Zod schema REJECTS object filter values. So `$ne` is a CLIENT-SIDE-only operator. Fix (all in `packages/core/src/data/store-api.ts`): toObsParams drops `$ne` from the wire (sends only a co-located `$ctx` positive scope, if any); new `applyClientFilter` excludes rows post-fetch via the SSOT `matchesFilter` (store-filter.ts — same predicate ExternalStore uses); new `cacheKeyFor` folds the resolved `$ne` into the cache identity so `{$ne:'_T'}` and `{$ne:'R2'}` over the same wire slice don't collide. Regression: apiStore.async.test.ts "$ne exclusion" describe.

**Why / How to apply:** for any per-spec scope on the async store, verify BOTH the warm AND the read derive the SAME (wire params + client-side discriminant) cache key, and that a filter operator the WIRE ROUTE can't express is resolved client-side (never String()-ified onto the URL). The render probe's empty/error check passes while the chart is silently empty — caught only by tracing the wire filter param or reading screenshots.
