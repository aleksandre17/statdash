## 2. Synthesis — the vital few (fix the root, kill the class)

Three roots account for every actionable find above. Fixing each is ONE change-set, not N patches.

### Root A — the pure engine's `GrowthResolver` multi-code branch bypasses its own generic conventions  → **kills 1a + 1b in one edit**
`resolvers.ts:218-229` is the single hot spot. It hardcodes the `'time'` dim key (1a) AND a first-tenant color field `accountColor` (1b), while the *sibling single-code branch* in the same class does both correctly (`atTime`/`TIME_DIM`, and it defers color to encoding). This is the **highest-value fix**: it is a Law-1 breach living in `packages/core`, the one layer the platform's crown proof (`second-tenant.fitness`) exists to keep tenant-agnostic. One rewrite of that branch — route time through `atTime`, source color from the declared encoding channel — restores the file to its own stated invariant (`:29`) and removes the last privileged-name leak I could find in the engine. **Note (Lane B): `packages/core` was just edited/uncommitted this session — verify against the quiesced tree before touching.**

### Root B — `apps/api` has no shared home for boundary vocabulary  → **kills 2 + 4 with one discipline**
`LocaleString` (×4 copies) and the reserved `'measure'` dim-code (magic string ×3, with a module-private constant) are the **same disease**: api re-types/inlines shared vocabulary per-file instead of importing an SSOT. The arrow already blesses `apps/api ← packages/contracts` and api already uses it. Fix = collapse both to shared homes (`ContractLocaleString` in contracts; a `MEASURE_DIM` constant beside it or in an api `lib/dims.ts`). Encode the invariant as a tiny fitness test that fails on a new `= Record<string,string>` locale re-declaration in `apps/api/src/**`.

### Root C — capability is declared but not fully trusted as the SSOT  → **kills 3**
`capabilityGate.needsGeo()` hedges declared caps with type-name sniffing — the one spot in the otherwise-exemplary Constructor palette logic that re-privileges node types by literal, against its own header. Declare a `geo` capability and gate on it alone. This is the same "trust the registry/caps, never a literal type list" principle the entire platform rests on; the fix makes the palette fully capability-driven and removes a latent RX-16 coupling.

### Ranking rationale (blast-radius × future-pain)
1. **Root A (1a/1b)** — HIGH: Law-1 leak *inside the pure engine*, the most-guarded invariant; small edit, large principle. Do first.
2. **Root C (3)** — MED: self-contradicting capability gate on the authoring surface; small edit; prevents a whole "re-privilege by type-name" regression class if turned into a fitness (grep for `.type === '` / `.type.includes(` in `discovery/**`).
3. **Root B (2/4)** — MED/LOW: DRY hygiene with real shotgun-surgery latency; low urgency, high tidiness ROI; bundle as one api-vocabulary-SSOT change-set.
4. **5, 6** — LOW/INFO: 5 is allowlist-sanctioned SSOT tidy-up; 6 is a consistency observation, YAGNI-gated.
