# Pattern: I18n — LocaleString + Hooks

Full spec → `docs/architecture/subsystems/29-i18n-architecture.md`

## LocaleString

```ts
type LocaleString = string | Record<string, string>
// { ka: 'მშპ', en: 'GDP' }   ← JSON-serializable, Constructor-ready
// 'plain string'              ← backward compat ✅
```

## In Shell (inner component only)

```tsx
function SectionControl({ def, ctx }) {
  const t   = useResolveLocale()    // content strings — titles, labels, units
  const fmt = useFmt()              // number/date formatting
  const tSys = useT('section')      // system UI strings — buttons, tooltips

  return (
    <section>
      <h2>{t(def.title)}</h2>                           // LocaleString → string
      <span>{fmt.number(42_000, { decimals: 1 })}</span> // 42 000,0 (ka) / 42,000.0 (en)
      <button>{tSys('export')}</button>                  // 'ექსპორტი' | 'Export'
    </section>
  )
}
```

## In Table (classifiers from ctx)

```tsx
// classifiers pre-injected by renderNode step 3 — never call store directly in shell
{resolveLabel(String(row[col.key]), ctx.classifiers[col.key], ctx.locale, ctx.fallbackLocale)}
```

## Anti-patterns

```ts
❌  resolveLocaleString(def.title, ctx.locale, 'ka')   // hardcoded fallback
❌  fmt in RenderContext                                // function in data ctx = SRP violation
❌  shell calls store.classifiers directly             // ISP violation
✅  useResolveLocale() → t(def.title)
✅  useFmt() → fmt.number(v)
✅  ctx.classifiers (injected by engine)
```
